# Add Tests

This folder contains implementations that need test coverage.

## Test Scenarios

| Folder          | Implementation      | Test Status                         |
| --------------- | ------------------- | ----------------------------------- |
| `calculator/`   | Calculator class    | Partial tests - needs more coverage |
| `user-service/` | User CRUD service   | Empty test file                     |
| `string-utils/` | String manipulation | No tests                            |

## How to Test

1. Open an implementation file
2. Ask: "Write unit tests for this"
3. Open the corresponding test file
4. Ask: "Add more test cases for edge cases"

## Example Prompts

- "Generate comprehensive unit tests"
- "Add tests for edge cases"
- "Write tests using Jest"
- "Test the error handling"
- "Add integration tests"
- "What test cases are missing?"
